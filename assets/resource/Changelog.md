# v1.4.4 更新公告

> [!NOTE]
> 本版本新增 **事件保底选择机制**，优化了事件选择架构与培育流程稳定性，并修复了屏幕旋转检测等多项 Bug。

## 主要更新

- **新增卡片支持**：新增篠澤広卡片「ガラクタロード/荆棘之路」并设为默认选择
- **事件保底选择机制**：新增事件保底选择机制，当无法匹配优先事件时自动回退到可用选项，避免流程卡住
- **事件选择架构重构**：重构事件选择逻辑，提取基类 `ProduceChooseEventBase`，引入优先级系统，集中管理 `run_task` 映射
- **回忆已生成提示**：培育过程中新增"回忆已生成"日志提示，便于跟踪培育进度
- **模板匹配增强**：在删除/复制/变换选择界面增加模板匹配，提升界面识别准确性
- **第二属性课程扩展**：扩展第二属性课程触发条件，增加第一属性停止时的处理逻辑
- **培育流程优化**：优化培育流程中的延迟参数与匹配逻辑，提升运行效率
- **资源更新**：更新多项资源图片，新增培育主页面排行榜图标，移除旧的工作图标；更新 OK 按钮图标
- **日志优化**：优化日志颜色，统一使用更准确的颜色名称

## 问题修复

- 修复屏幕旋转检测时点击操作方法与顺序错误的问题
- 修复 OK 按钮图标不匹配的问题

> [!IMPORTANT]
> 受第三方UI影响，须在`MAG` - `设置` - `运行设置` 中 关闭 `实时视图` 选项
>

> [!IMPORTANT]
> 使用 DMM 端时，需要以管理员身份启动程序。
